# Ozon MCP - Функциональные возможности

Текущая версия: 1.0.0
Основано на: Blueprint MCP (refactored for Ozon)

Этот документ описывает все актуальные возможности сервера, включая специализированные инструменты для Ozon и базовые функции управления браузером.

---

## 🛒 Ozon-специфичные инструменты

Специализированные инструменты, оптимизированные для работы с маркетплейсом Ozon. Логика работы основана на надежных CSS-селекторах (настраиваются в `selectors/ozon-selectors.json`) и JS-инъекциях.

### `ozon_search_and_parse`
**Поиск и парсинг товаров**
- **Назначение:** Полный цикл поиска товаров по запросу.
- **Действие:** 
  1. Вводит поисковой запрос в строку поиска.
  2. Ожидает загрузки результатов (grid).
  3. Парсит результаты выдачи через JS, извлекая:
     - Заголовок товара.
     - Цену (текущую).
     - Ссылку на товар.
     - Индекс в выдаче.
- **Output:** JSON массив с найденными товарами.

### `ozon_parse_product_page`
**Парсинг карточки товара**
- **Назначение:** Получение детальной информации с *уже открытой* страницы товара.
- **Действие:**
  1. Проверяет, что текущая страница - это карточка товара.
  2. Извлекает:
     - Название (H1).
     - Цену (текущий Main Price).
     - Доступные вариации (цвета/размеры/конфигурации).
     - Статус наличия ("Available").
- **Output:** JSON объект с деталями товара.

### `ozon_cart_action`
**Умное управление корзиной**
- **Назначение:** Добавление товаров или изменение их количества.
- **Действия (`action`):** 
  - `add`: Добавить товар (нажать кнопку "В корзину"). Если уже в корзине — ничего не делает или уведомляет.
  - `increment`: Увеличить количество (+).
  - `decrement`: Уменьшить количество (-).
- **Логика:**
  - Автоматически определяет состояние кнопки (текст "В корзину" или счетчик количества).
  - Валидирует результат действия (проверяет счетчик товаров в хедере сайта).

### `ozon_get_share_link`
**Получение чистой ссылки**
- **Назначение:** Извлечение канонической ссылки без мусорных параметров.
- **Алгоритм:**
  1. Ищет `<link rel='canonical'>`.
  2. Если нет, ищет OpenGraph тег.
  3. В крайнем случае берет текущий URL и отрезает Query Parameters.

### `ozon_ownership_status`
**Статус эксклюзивного доступа**
- **Назначение:** Проверка статуса владения браузерным мостом данным экземпляром MCP.
- **Output:** JSON объект со статусом lease.

## 🌐 Управление Браузером (Core)

Базовые возможности управления реальным Chrome браузером через DevTools Protocol. Полностью наследует функционал Blueprint MCP (Local Transport).

### Вкладки (`browser_tabs`)
- `list` — список всех открытых вкладок с заголовками и URL.
- `new` — открытие новой пустой вкладки или сразу с URL.
- `attach` — подключение к вкладке для управления (необходимо перед взаимодействием).
- `close` — закрытие указанной вкладки.

### Навигация (`browser_navigate`)
- `url` — переход по прямой ссылке.
- `back` / `forward` — навигация по истории браузера.
- `reload` — перезагрузка текущей страницы.

### Взаимодействие (`browser_interact`)
Универсальный инструмент для эмуляции действий пользователя через последовательность шагов:
- **Click:** Клик по CSS-селектору. Поддерживает `clickCount` (двойной/тройной клик) и выбор кнопки мыши (`left`, `right`, `middle`).
- **Type:** Ввод текста в инпут по селектору.
- **Clear:** Очистка содержимого инпута.
- **Press Key:** Эмуляция нажатия физических клавиш (Enter, Tab, Escape, Arrows).
- **Scroll by:** Прокрутка страницы на указанное количество пикселей.
- **Scroll into view:** Прокрутка элемента в зону видимости.
- **Wait:** Пауза (timeout в мс) для ожидания асинхронных событий.
- **Hover:** Эмуляция наведения курсора мыши.
- **Sequence:** Поддержка выполнения цепочки действий с обработкой ошибок (`stop` или `ignore`).

## 📄 Работа с контентом и Анализ

### Контент страницы
- **`browser_snapshot`**: Получение Accessibility Tree страницы (текстовая структура для LLM).
- **`browser_evaluate`**: Исполнение произвольного JavaScript кода в контексте страницы. Возвращает результат выполнения.

### Визуализация
- **`browser_take_screenshot`**: Снятие скриншота.
  - Full Page или Viewport.
  - Скриншот конкретного элемента по селектору.
  - Сохранение в файл или возрат base64.

### Отладка
- **`browser_console_messages`**: Получение логов из консоли браузера (сбор или очистка буфера сообщений).
- **`browser_handle_dialog`**: Принятие или отклонение активных JS-диалогов (alert, confirm, prompt).

## 🛡️ Архитектура и Безопасность

### Local Direct Transport
В отличие от оригинального Blueprint MCP, версия Ozon MCP работает **только в локальном режиме**:
- **Нет облачного прокси:** Все взаимодействие происходит напрямую между MCP сервером и расширением.
- **WebSocket:** Локальный сервер поднимается на порту `5555` и ждет подключения расширения.
- **Безопасность данных:** Куки и сессии никогда не покидают ваш компьютер.

### Stealth Mode
Встроенный режим маскировки через параметр `stealth` в `browser_tabs`:
- Позволяет обходить базовые проверки на автоматизацию (Bot Detection).
- Использует реальный User Profile браузера Chrome, сохраняя историю посещений и авторизацию в Ozon.

### Ownership Arbiter (Эксклюзивный доступ)
- Механизм управления эксклюзивным доступом к браузерному мосту.
- Возвращает статус через `ozon_ownership_status`.
